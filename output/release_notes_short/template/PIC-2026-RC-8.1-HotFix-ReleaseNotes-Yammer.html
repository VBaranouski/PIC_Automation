PICASso Hot Fix — PIC-2026-RC-8.1

Release Date: February 18, 2026  •  Total Fixes: 21
We are pleased to announce the successful deployment of the PICASso 8.1 Hot Fix release. This update addresses critical and high-priority defects affecting action management, reviewer assignment logic, release cloning, user role visibility, and BU Security Officer FCSR date handling. Several UI misspellings and data consistency issues are also resolved.

High Priority Fixes

5 items
  • Fixed an issue where a user who opened the FCSR Review tab, clicked Edit and Save without making any changes was incorrectly recorded as “Updated by” for all actions on the page.
  • Fixed an issue where users could not create new actions on a release if any existing closed/non-editable action had empty mandatory fields.
  • Fixed an issue where updating Risk Classification, Data Protection & Privacy Risk, or Questionnaire incorrectly triggered a Requirements Versioning pop-up even though no version changes were made in Backoffice.
  • Fixed an issue where the “Clone from existing release” dropdown included inactive releases, which caused the cloned release to not appear in the UI after creation.
  • Fixed an issue where the Product Admin Modification role was assigned to users who did not create the product.

Medium Priority Fixes

12 items
  • Fixed an issue where changing the “Last BU Security Officer FCSR Date” for a release did not trigger recalculation of assigned reviewers for Scope Approval and FCSR Review stages.
  • Fixed an issue where the “Last BU Security Officer FCSR Date” was outdated for existing production releases and products, causing incorrect reviewer assignments for in-flight releases.
  • Fixed an issue where the “Last BU Security Officer FCSR Date” was being overwritten for completed, cancelled, and inactive releases whenever the date changed on another release.
  • Fixed an issue where the “Last BU Security Officer FCSR Date” on Release Details was overridden by updates made on Product Details, causing the release to display an incorrect date.
  • Fixed an issue where users with the LOB Security Leader role could see all releases in PICASso regardless of their assigned organizational scope.
  • Fixed an issue where the “LOB Security Leader” role was disappearing from the release sidebar after all approval stages were completed.
  • Fixed an issue where BU Security Officer was not assigned as a reviewer for releases where the “Last BU Security Officer FCSR Date” was empty or older than 12 months.
  • Fixed an issue where the GetProductRelease API response did not include the Release Creation Date field.
  • Fixed an issue where the “Updated by” and “Last Update” fields showed inconsistent values between the Action tab on the release and the Action Management view.
  • Fixed an issue where applying a filter with no results on the My DOCs tab caused the entire filter panel to disappear, requiring a full page refresh to restore it.
  • Fixed misspellings in success and info messages: “Sucessfully” → “Successfully” and “Requeriment” → “Requirement”.
  • Fixed an issue where users could not reopen an action that was closed with incorrect data.

Low Priority Fixes

2 items
  • Fixed an issue where the Closure Comment field was not cleared after an action’s status was changed away from “Closed.”
  • Fixed an issue where the Generate Report warning displayed “atleast” instead of “at least.”
