## Tiers

| Tier | Purpose | Loaded |
|---|---|---|
| 1 | Compact always-in-context routing & recent-change awareness | Every conversation |
| 2 | Per-area feature catalog + per-topic distilled knowledge | 1–2 files per task |
| 3 | Metadata-only corpus indexes (title/id/version/heading). Never loaded in bulk — grep only. | Surgical `grep_search` |
| 4 | Runtime fetches: `confluence_get_page`, `jira_get_issue`, `npm run tracker:list`, surgical `read_file` with line range | On demand |

See [.github/instructions/knowledge-routing.instructions.md](../../../.github/instructions/knowledge-routing.instructions.md) for the full agent decision tree.

## Regenerating indexes

```bash
npm run index:all             # all corpus indexes (scenario + user-guide + confluence)
npm run index:scenarios       # tracker DB → corpus/scenario-index.md
npm run index:user-guide      # UG .md files → corpus/user-guide-index.md
npm run index:confluence      # Confluence REST → corpus/confluence-index.md (requires PAT)
npm run index:jira-stories    # Jira stories → corpus/jira-stories-index.md
npm run index:jira-tests      # Jira tests   → corpus/jira-tests-index.md
npm run sync:knowledge -- --area <area>   # re-sync stale pages for one area
```

One-off utility: `node scripts/fetch-confluence-bodies.mjs <pageId> [...]` — fetches and strips HTML from specific Confluence pages for distillation into `knowledge/*.md` files.

## QA exploration walk

`node scripts/explore-qa.mjs` — headless Chromium walk of the QA environment. Logs in with the PQL default user, probes 5 landing tabs + Product/Release/DOC detail pages via tab-scoped DOM capture, and writes `exploration-findings.md` (Tier 3, read on-demand). Run this after significant UI changes.

## Classification rule (NEW vs IMPROVEMENT)

1. Read [feature-areas.md](feature-areas.md) (already in context). Keyword-match the task.
2. Open the matching `feature-registry/<area>.md`.
3. Keyword-match the task against existing feature-ids.
4. **Match ≥ 1 feature-id → IMPROVEMENT.** Reuse scenario-id prefix; check tracker with `npm run tracker:list -- -a <area> -s <keyword>`.
5. **No match** (also grep `corpus/*-index.md`) → **NEW.** Mint `<area>.<slug>`, add registry row, add change-impact row, seed scenario-ids.

## How to add a new knowledge topic

1. Copy `knowledge/_template.md` → `knowledge/<topic>.md`.
2. Fill the template using:
   - Confluence page body via `node scripts/fetch-confluence-bodies.mjs <pageId>`.
   - UG section via `read_file(startLine, endLine)` using the anchor from `corpus/user-guide-index.md`.
   - QA DOM from `exploration-findings.md`.
3. Add/update the feature-id row in `feature-registry/<area>.md` with `last-verified` date.
4. Add a row to `change-impact.md` (type `NEW` or `IMPROVEMENT`, status `pending`).
5. Add the topic slug to the matching `knowledge topics` cell in `feature-areas.md`.
