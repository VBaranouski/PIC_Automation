# Knowledge Base

Token-efficient documentation and context store for the PICASso automation agent.

## Layout

```
docs/ai/knowledge-base/
├── README.md                        ← this file
├── feature-areas.md                 ← Tier 1 · always loaded · ~20 rows
├── change-impact.md                 ← Tier 1 · last 2 sprints · rolled
├── change-impact-archive.md         ← archive of rolled-out entries
├── .sync-state.json                 ← last-fetched versions per Confluence page
├── feature-registry/                ← Tier 2 · on-demand per area
│   ├── _template.md
│   ├── auth.md
│   ├── landing.md
│   ├── products.md
│   ├── releases.md
│   ├── doc.md
│   ├── reports.md
│   ├── backoffice.md
│   ├── integrations.md
│   └── other.md
├── knowledge/                       ← Tier 2 · on-demand per topic
│   └── _template.md
└── corpus/                          ← Tier 3 · metadata indexes only
    ├── README.md
    ├── confluence-index.md          ← generated by scripts/index-confluence.ts
    ├── user-guide-index.md          ← generated by scripts/index-user-guide.ts
    └── scenario-index.md            ← generated by scripts/index-scenarios.ts
```

## Tiers

| Tier | Purpose | Loaded |
|---|---|---|
| 1 | Compact always-in-context routing & recent-change awareness | Every conversation |
| 2 | Per-area feature catalog + per-topic distilled knowledge | 1–2 files per task |
| 3 | Metadata-only corpus indexes (title/id/version). Content never loaded in bulk. | Greppable only |
| 4 | Runtime surgical fetches via MCP Atlassian, tracker CLI, grep | On demand |

See [.github/instructions/knowledge-routing.instructions.md](../../../.github/instructions/knowledge-routing.instructions.md) for the agent decision tree.

## Regenerating indexes

```bash
npm run index:all             # all three corpus indexes
npm run index:scenarios       # tracker DB → scenario-index.md
npm run index:user-guide      # user-guide.md TOC → user-guide-index.md
npm run index:confluence      # Confluence REST → confluence-index.md (requires JIRA/CONFLUENCE PAT)
npm run sync:knowledge -- --area <area>
```

## Classification rule (NEW vs IMPROVEMENT)

1. Read [feature-areas.md](feature-areas.md) (already in context).
2. Open the matching `feature-registry/<area>.md`.
3. Keyword-match the task against feature-ids in that file.
4. **Match ≥ 1 feature-id → IMPROVEMENT.** Reuse scenario-id prefix; check tracker for existing coverage.
5. **No match** (after also greping `corpus/*-index.md`) → **NEW.** Mint a feature-id (`<area>.<slug>`), add a registry row, seed scenario-ids.
