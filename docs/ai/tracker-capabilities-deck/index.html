The Problem

Why Existing Test Cases Can't Drive Automation

PICASso is a complex GRC platform built on OutSystems with 78 roles, 157 privileges, and multi-stage document lifecycle workflows spanning products, releases, questionnaires, requirements, and more. It includes deep integrations with third-party systems — JAMA, Jira, and SAST tools like BDBA — as well as its own Ingestion and Extraction APIs and complex back-office business logic. The existing Xray/Jira test inventory was built for manual testers and fundamentally cannot be used for automation as-is.

error 4,220+ Xray Test Cases

~50%
are outdated and require updates before any use
  • Written for manual testers — steps like "verify the page looks correct" are untestable by code
  • No structured locator hints, no DOM references, no automation-ready assertions
  • Many reference features or flows that have been redesigned since writing
  • Missing important automation steps, or expected results are too broad to assert programmatically
  • Some test cases are too generic — verifying multiple scenarios at once, violating the automation rule: one test case = one check

What It Is

Not Test Case Storage — An Automation Command Center

The tracker is not a replacement for Xray or a generic test case repository. It is a purpose-built operational platform that stores automation-ready scenarios and tracks their lifecycle from pending through automated, executed, and reported — all the way to Jira defect submission.

track_changes

Automation Lifecycle Tracking

Every scenario has a dual-status model: automation state (pending → automated → on-hold) and execution status (passed / failed / skipped). Know exactly what's been automated, what's pending, and what's broken — at a glance.

play_circle

One-Click Test Execution

Run any scenario, subsection, or full spec directly from the dashboard. Live terminal output streams to the built-in panel. No context-switching to terminal or IDE required.

bar_chart

Instant Report Navigation

After each run, Allure reports are auto-generated and served via a single sidebar link. View detailed pass/fail breakdown, failure screenshots, execution traces — without manual report generation steps.

bug_report

One-Click Jira Submission

Found a defect? The Allure report contains full failure context — screenshots, trace, assertion details. Submit a Jira ticket with all the information pre-populated from the report in one click. No manual copy-paste of error details.

sync

Automatic Status Sync

After each test run, the tracker parses Playwright's JSON report and updates every matched scenario's execution status. Pass, fail, skip, and timeout are detected automatically — no manual status updates needed.

import_export

Export & Import of Test Cases

Export scenarios to Excel for review in a familiar format. Import updated or new scenarios from Excel — workflow-scoped, validated, and deduplicated automatically.

Transformation

Before vs. After

The shift from manual test management to an AI-driven automation pipeline.

Before — Manual World

  • 4,220+ Xray test cases written for manual execution — unusable for automation scripts
  • ~50% of test cases outdated with no systematic way to identify which ones
  • No visibility into automation coverage — "which scenarios actually have scripts?"
  • Test results communicated via email or chat — no single source of truth
  • Defect submission requires manual screenshots, copy-paste of error logs to Jira
  • Engineers rediscover which tests cover which features every sprint

After — Tracker + AI Pipeline

  • AI analyzes Xray cases, updates outdated steps, migrates into automation-ready format
  • Every scenario validated, deduplicated, and linked to its executable test code
  • Real-time dashboard shows automation state, execution status, and priority per scenario
  • One-click: run the test or suite from the UI → view Allure report with results
  • AI writes test scripts following best practices, then self-validates via Playwright MCP
  • Failed tests are marked in the tracker — one click creates a Jira defect with all steps to reproduce and screenshots attached
