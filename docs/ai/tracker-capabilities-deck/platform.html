Technical Deep Dive

Platform & Architecture

A lightweight, zero-cloud-dependency stack purpose-built for PICASso automation operations — from the SQLite data layer through the REST API to the dark-mode dashboard and CLI tooling.

Architecture

System Layers

Each layer is independently testable and scriptable. The entire stack starts in under 2 seconds.

Frontend
Single-page dark-mode dashboard — Tailwind CSS + vanilla JS. Search, filters, workflow accordion, detail drawer, run dialog, XLSX modal.
REST API
Express 5 server with 30+ endpoints — CRUD, status shortcuts, run control, stats, sync, import/export, Allure data, project resolution.
Tracker Core
Modular TypeScript library: crud.ts, sync.ts, stats.ts, import-export.ts, spec-parser.ts, helpers.ts, migrations.ts.
Database
SQLite via better-sqlite3 — WAL mode, foreign keys, versioned schema with migration support. Tables: scenarios, scenario_details, scenario_groups, groups, scenario_merges, meta.
Test Layer
Playwright + TypeScript — 15+ named projects with dependency chains. Four-layer architecture: locators → page objects → fixtures → tests.
CLI
npm run tracker — list, sync, hold, group, export, import. Scriptable for CI pipelines and batch operations.

Every tracker operation is exposed as a REST call — enabling CI integration, external tooling, and AI agent automation.

EndpointPurpose

DELETE /api/scenarios/:idRemove scenario with cascading group and detail cleanup
POST /api/scenarios/:id/auto-stateSet automation lifecycle state (pending → automated → on-hold → updated)
POST /api/scenarios/:id/exec-statusSet execution result (passed, not-executed, skipped, failed-defect)
POST /api/runStart a Playwright test run — spec files, project, grep, headed mode
GET /api/run/statusPoll run state with incremental line output (supports since-based streaming)
POST /api/run/cancelCancel an active test run with graceful SIGTERM → SIGKILL fallback
POST /api/run/sync-resultsSync scenario statuses from the latest Playwright JSON report
GET /api/run/historyRecent test run history with duration, summary, and status

Technical Differentiators

Why This, Not Off-the-Shelf

Generic test management tools lack the deep OutSystems integration, real-time execution coupling, and AI-augmented workflow that PICASso demands.

precision_manufacturing

OutSystems-Native Patterns

Built-in handling of OSUI custom dropdowns, partial-refresh DOM replacement, CSS-generated asterisks, user lookup widgets, and cascading select elements — none of which generic tools understand.

psychology

AI-Assisted Test Generation

The create-auto-tests skill integrates AI agents for scenario analysis, test code generation, Playwright MCP browser validation, and self-correcting locator fixes — with human-in-the-loop verification at every stage.

speed

Zero-Latency Local Stack

No cloud round-trips. SQLite provides sub-millisecond queries. The entire tracker starts in under 2 seconds. Tests execute on local Chromium with full DOM access for debugging.

security

Role-Aware Test Design

Test scenarios are mapped to PICASso's 78 roles and 157 privileges. The tracker stores role codes and access-based preconditions alongside each scenario for precise coverage of permission boundaries.
